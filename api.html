API

Access all 11,772 SEC litigation cases programmatically via our REST API. Perfect for research, analysis, and building applications.

Quick Start

Start the API server locally to access all cases:

# Install dependencies pip install -r requirements.txt # Start the server python api_server.py # Server runs on http://localhost:5000

Note: The API loads cases from litigation-cases.json on first request and caches them in memory for fast performance.

Endpoints

GET /api/metadata

Get metadata about the dataset, including total case count and scrape date.

Example Response

{ "scrapedAt": "2025-12-18T02:11:23.296Z", "totalCases": 11772, "source": "https://www.sec.gov/..." }
GET /api/cases

Get all cases with pagination and optional date filtering.

Query Parameters

page int
Page number (default: 1)
per_page int
Items per page (default: 100, max: 1000)
release_date_from string
Filter cases from this date (YYYY-MM-DD)
release_date_to string
Filter cases to this date (YYYY-MM-DD)

Example Request

GET /api/cases?page=1&per_page=100 GET /api/cases?release_date_from=2024-01-01&release_date_to=2024-12-31
GET /api/cases/<release_number>

Get a specific case by release number (e.g., "LR-26445" or "26445").

Example Request

GET /api/cases/LR-26445 GET /api/cases/26445
GET /api/cases/search

Search cases by various criteria including text search, court, charges, and complaint availability.

Query Parameters

q string
Text search in title and fullText
title string
Filter by title (partial match)
court string
Filter by court name
charges string
Filter by charges (partial match)
has_complaint boolean
Filter cases that have complaint PDFs (true/false)
page int
Page number (default: 1)
per_page int
Items per page (default: 100, max: 1000)

Example Requests

GET /api/cases/search?q=fraud&has_complaint=true GET /api/cases/search?court=California&page=1 GET /api/cases/search?q=fraud&charges=Section%2010(b)
GET /api/health

Health check endpoint to verify the API is running and cases are loaded.

Usage Examples

Python

import requests BASE_URL = "http://localhost:5000/api" # Get all cases (paginated) response = requests.get(f"{BASE_URL}/cases", params={"page": 1, "per_page": 100}) data = response.json() print(f"Total cases: {data['total']}") # Get a specific case case = requests.get(f"{BASE_URL}/cases/LR-26445").json() print(case['title']) # Search cases results = requests.get( f"{BASE_URL}/cases/search", params={"q": "fraud", "has_complaint": "true", "page": 1} ).json() print(f"Found {results['total']} cases")

cURL

# Get first page of cases curl "http://localhost:5000/api/cases?page=1&per_page=100" # Get specific case curl "http://localhost:5000/api/cases/LR-26445" # Search for fraud cases curl "http://localhost:5000/api/cases/search?q=fraud&has_complaint=true"

Download All Cases

Use the provided example script to download all 11,772 cases:

python api_example.py

Or programmatically:

from api_example import download_all_cases download_all_cases("all_cases.json")

Case Data Structure

Each case contains the following fields:

{ "releaseNumber": "LR-26445", "releaseDate": "2025-12-16", "title": "Case Title", "url": "https://www.sec.gov/...", "features": { "summary": "...", "caseName": "...", "respondents": "...", "charges": "...", "court": "...", "fullText": "...", "originalSummary": "..." }, "supportingDocuments": [ { "title": "SEC Complaint", "url": "https://www.sec.gov/files/...", "type": "complaint" } ] }
